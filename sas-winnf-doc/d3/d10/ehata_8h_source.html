16 // ******* WinnForum extension *******
17 // Change all inputs and intermediate value from float to double
18 // The reason for that are:
19 // - to make the interface more in line with the ITM model
20 // - to suppress all mismatch between different implementations:
21 // By using float, every implementation will lead to slightly different
22 // results depending on the compiler and manual code optimisation done.
23 //
24 // Because the propagation model is strongly non linear, this typically
25 // leads to variation which can exceed a fraction of a dB or more.
26 // By using double, this issue is strongly alleviated (with the appropriate
27 // compiler flag to force exactness of double calculation, for example the
28 // -fno-unsafe-math-optimizations in gcc or the /fp:precise (default) in
29 // Microsoft compiler).
30 //
31 // Note: Hacky but works perfect if ehata.h last #include in the C++ files.
32 #define float double

38 // Function to get the distance in meters from a profile:
39 // the `pfl` profile store the number of points and the step between
